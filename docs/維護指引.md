# 閩拚輸入法維護操作指引

## 降低輸入漢字預估精準度

### 問題摘要

中州韻（RIME）的小狼毫輸入法，會隨著使用者的操作使用次數，逐次改善【推估預測】精準度，並配合【候選字視窗】之顯示，將使用者可能欲輸入之漢字放在最上頭。此功能本是好事；但使用者若想要模擬【閩拚輸入法】，剛開始操作使用之【初始狀態】，則此貼心的功能，反成了阻礙；因為，使用者才剛輸入拼音字母，欲輸入之【漢字】，即已顯示在【候選字視窗】之第一列。

以下說明，欲改善此問題之操作。

### 解法指引

小狼毫目前會依詞頻自動把【常用字】排在第一位。此問題之解法有以下兩種方向：

1. 透過設定，降低預測精準度；
2. 透過【清除使用者詞頻】回到初始狀態。

#### 1. 透過設定，降低預測精準度

弱化即時預測：在【檔案】： %APPDATA%\Rime\weasel.custom.yaml（若找不到檔案，則自行手動建立），加入以下設定：

```YAML
patch:  
  "translator/enable_completion": false      # 關掉未輸入完就補字
  "translator/enable_sentence": false        # 不組長句
  "speller/auto_select": false               # 禁止自動選第一候選
  "translator/enable_user_dict": false       # 不參考使用者詞典（可視需求再打開）
```

存檔後按 Ctrl+\` （或點系統列圖示選「重新部署」）讓設定生效。也可只針對某個方案在 schema_name.custom.yaml` 內 patch，同樣鍵值。

#### 2. 透過【清除使用者詞頻】回到初始狀態

回到「初始詞頻」：

- 備份 %APPDATA%\Rime 資料夾後，刪除或移走其中的 .userdb、build/、installation.yaml 等檔案；
- 重新部署，RIME 會依原始詞庫重建，所有學習過的頻率會被清零。
- 若只想針對某【方案】（方案指：某輸入法），刪除對應 xxx.userdb 即可。

日後維護：

- 想要偶爾清空，可寫批次腳本刪除 .userdb 再呼叫 WeaselDeployer.exe /deploy；
- 或在輸入法選單中用「輸入法設定→資料維護→重新部署」手動完成。

如需同時保持自訂詞條又不讓它自動置頂，可以僅將 speller/auto_select 設為 false，並保留使用者詞典。
